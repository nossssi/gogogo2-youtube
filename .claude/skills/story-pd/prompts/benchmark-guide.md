# 딥 벤치마킹 — 공식 개정 후보 수집 (편당 필수 아님)

**목적: 아웃라인 재료가 아니라 playbook(script-guide.md) 개정 후보 수집.** 매 편 대본은 공식 현재 버전을 그대로 쓴다.

**트리거** (이때만 실행): ① 신규 유망 채널 발견 ② 내 영상이 기대 이하 ③ 5~10편마다 정기 점검. 사용자가 "벤치마킹 하자"고 하면 이 문서를 로드.

## 절차

1. **대상 선정** — 최신 스캔 리포트(`channels/{채널}/research/scan_*.md`)에서 조회수/일 상위 3~5편, 또는 사용자 지정 URL.
2. **수집** — 자막(=대본 전문)·메타·댓글 TOP10·썸네일:
   ```
   python3 scripts/research/collect_refs.py --channel {채널} --benchmark URL1 URL2 ...
   ```
   → `channels/{채널}/research/refs/{NNN}/` (meta.md, transcript.txt, thumbnail.webp)
3. **ref별 구조 청사진** — transcript를 읽고 편마다 작성:
   ```
   ■ 콜드 오픈: 3요소(충격 장면/결말 흘리기/질문) 충족 여부 + 실제 문장
   ■ 브릿지: 채널 루틴·되감기 방식
   ■ 막 비율: 결핍 X% / 사건 X% / 갈등 X% / 파국 X% / 반전·응보 X% / 해소 X% (글자수 기준)
   ■ 리텐션: 후렴 N회(위치 %), 떡밥 리마인드 N회, 시점교차·패턴인터럽트 유무
   ■ 복선: 개수, 심는·회수 위치, 토씨 재사용 여부, 이중 반전 유무
   ■ 감정축·장르: 주축 + 양념 / 황금률(직접 복수 금지) 준수 여부
   ■ 화법: 종결어미 분포, 대사 직접/간접 비율, 문장 길이
   ■ 분량·챕터: 총 글자수, 장 구분 방식, 장당 분량
   ■ 댓글 반응 포인트: 시청자가 반복 언급하는 요소 (몇 분 지점, 어떤 장면/정서)
   ```
4. **교차 비교 → 개정 제안서** `channels/{채널}/research/proposal_YYYYMMDD.md`:
   | 현행 규칙 (또는 공백) | 관찰 | 근거 강도 (채널 N곳 / 자기 실측) | 제안 (개정/유지/보류) |
   |---|---|---|---|

   `playbook-history.md`의 **개정 문턱**(채널 2곳+ 반복 또는 자기 실측)을 못 넘는 관찰은 **보류**로 남긴다.
5. **사용자 승인** → 승인된 항목만 `script-guide.md`/`script-constraints.md` 개정 + `playbook-history.md`에 v+1 기록(근거 포함). 승인 없으면 proposal로 종료.

## 자기 채널 실측 (업로드가 쌓이면 최우선 근거)

- 업로드 결과(조회수, 시청 지속 그래프의 이탈 지점)를 proposal의 근거 열에 채운다.
- 이탈 지점 ↔ 대본 위치(글자 %) 매핑: 어떤 장 경계·비트에서 빠졌는지 → 해당 규칙 개정 후보.
