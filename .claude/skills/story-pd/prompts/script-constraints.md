# 대본 제작 제약·검수 — DRAFT·REVIEW가 로드

파이프라인 호환 제약이다. **어기면 뒷단계(TTS·자막·스토리보드)가 고생한다.** 기계 검증 가능한 항목은 `scripts/script/validate_script.py`가 잡는다.

## 제작 제약

- **자막 줄 기준 ~20자**: 한 호흡(쉼표 구간)이 공백 제외 20자를 넘지 않게. vrew 모드는 클립이 곧 자막 줄이다.
- **아라비아 숫자·영어 금지**: "3년" → "삼 년". 지키면 tts_map 변환 단계를 통째로 생략한다.
- **따옴표 안 구두점**: 문장 분리기가 따옴표 안의 마침표로는 문장을 나누지 않는다. 대사가 여러 문장이면 따옴표를 문장별로 나눠 닫는 것을 우선 고려.
- **문단 = 씬 후보**: 빈 줄로 나눈 문단이 스토리보드 씬 분할의 단위가 된다. 단편은 20~30문단, 장편은 **장당 10~14문단** (장편의 씬 분할은 STORYBOARD에서 문단을 묶어서 한다 — 문단당 1씬이 아님). 각 막(장)에 그림이 되는 순간(시각적 이미지)을 최소 2개씩.

## 분량표 (공백 제외, 실측 약 240자/분 — Vrew 낭독)

| 목표 영상 | 글자수 |
|---|---|
| 5분 | ~1,200 |
| 7분 | ~1,700 |
| 10분 | ~2,400 |
| 125분 | ~30,000 |
| 145분 | ~35,000 |
| 165분 | ~40,000 |

장편 장(章)당 2,500~3,500자 ≈ 10~15분.

## 기계 검증 (REVIEW에서 필수 실행)

```
python3 scripts/script/validate_script.py {P}/script.txt --target 30000,40000        # 완성본
python3 scripts/script/validate_script.py {P}/_script/chapters --chapter-target 2500,3500   # 집필 중 장별
```
검사 항목: 공백 제외 글자수(목표 범위), 아라비아 숫자/로마자 0건, 20자 초과 호흡, 문어체 어미(였다/하였다/거늘/허나), 문단 수. **위반 0이 될 때까지 다음 단계 진행 금지.**

## 검수 체크리스트 (REVIEW — 기계 검증 통과 후 사람/PD 확인)

1. **연속성**: 인물 이름·호칭 일관 / 시간선 역행 없음 / outline 복선 장부의 전 항목 회수 완료.
2. **미끼 회수**: 콜드 오픈에서 던진 질문에 본문이 답했는가. 마지막 문장이 수미상관/주제문 봉인인가.
3. **감정축 수렴**: 절정 디테일과 마무리가 선언한 감정축 하나를 향하는가.
4. **문체 샘플링**: 각 장의 첫·끝 문단을 소리 내 읽듯 확인 — 경어체 구술 톤, 3문장 연속 같은 길이 없음, "그는" 연발 없음.
5. **장 경계**(장편): 모든 장이 후렴/클리프행어로 닫히는가, 같은 유형 연속 없음, 중반 재점화 존재.
6. **인물·장소 인계**: outline의 인물·장소 표가 본문과 일치하는가 (ASSET_EXTRACT가 이 표를 받아 characters.json을 만든다).

## meta.txt 규약 (대본과 함께 작성)

```
[제목]
<검색 노출용 롱테일 제목 — 상황+역설/반전 예고, 40~60자. "충격 상황 + 알고 보니 ~" 구조>

[썸네일 문구]
<2줄, 줄당 2~8자 — 궁금증 유발, 스포 금지>

[설명문]
<3문장 요약 (스포 직전까지) + 해시태그 줄 (#야담 #민담 … 10개 내외)>

[제작 메모]
- 감정축: … / 장르: … / 정체·반전: … / 길이: 약 N분
- 훅: (콜드 오픈에 던진 미끼 한 줄)
```

업로드 스크립트가 youtube.md 없을 때 이 파일에서 제목/설명/태그를 자동 추출한다 — 형식을 지킬 것.
