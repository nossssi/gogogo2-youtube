# 야담 대본 공식 (playbook) — v2

SCRIPT 단계의 CONCEPT·OUTLINE·DRAFT가 로드하는 **공식 본체**. 외부 참조 없이 이 문서만으로 대본을 쓴다.
잘 터진 야담 장편 5편(조회 10만+)의 공통 골격을 역추출한 규칙이다 — 주제는 매번 다르지만 **뼈대는 거의 동일**했다. 그 뼈대에 이번 주제를 얹는 것이 이 단계의 일이다.

- 제작 제약(TTS·자막 호환)·검수 체크리스트·meta.txt 규약 → `script-constraints.md` (DRAFT·REVIEW가 로드)
- 규칙별 근거(어느 채널·영상에서 왔나)·개정 이력·개정 문턱 → `playbook-history.md` (개정할 때만 로드)
- **이 공식은 매 편 바뀌지 않는다.** 개정은 딥 벤치마킹(`benchmark-guide.md`)이 개정 문턱을 넘고 사용자가 승인할 때만.

## 0. 먼저 정하는 것 — 감정축 & 장르 (대본당 하나)

제일 먼저 이 대본이 시청자에게 남길 **핵심 감정 하나**를 고른다. 나머지 모든 선택(훅·절정·결말)이 여기로 수렴한다.

| 감정축 | 장르 예 | 실제 흥행 사례 |
|---|---|---|
| **통쾌(사이다)** ★주력 | 권선징악·신분역전·응보 | 미친 며느리가 옳았음 증명 / 쫓겨난 며느리의 경제 복수 / 화상 흉터 며느리의 목장 제국 / 씨받이 누명의 유서 반전 |
| **눈물(감동)** | 보은·모정·치유·재회 | 폐가 속 요절 천재의 유산으로 굳은 손이 되살아남 |
| **소름(미스터리)** | 괴담·비밀·정체추적 | (단독보다 통쾌·눈물의 **양념**으로 쓰일 때 강함 — 흥행작 중 순수 공포 없음) |

- **흥행작 다수가 "통쾌"가 주축, "눈물"은 마무리 양념.** 순수 공포/괴담은 없었다 — "귀신 폐가"조차 정체는 유산(감동)이었다. **괴담 제목으로 낚고 감동/응보를 먹이는 미끼바꿈**이 실제 패턴.
- **황금률: 주인공은 직접 복수하지 않는다.** 하늘·관아·악인의 자멸이 응징하고, 주인공은 끝까지 도덕적으로 깨끗하다. (씨받이 편: "제가 갚을 원한이 아닙니다. 하늘이 이미 갚았으니까요.") 이게 통쾌함을 "정의"로 격상시킨다.
- meta.txt 제작 메모에 감정축·장르·정체를 기록.

## 1. 흥행 골격 (길이 무관)

```
콜드 오픈(미끼)  →  브릿지(되감기)  →  본문 아크(결핍→사건→갈등 상승→파국→반전→응보)  →  다중 해소 + 교훈 봉인  →  채널 CTA
```

이 순서를 그대로 따른다. 아래 각 블록의 공식을 채운다.

### ① 콜드 오픈 (첫 3~5문장, 약 1분) — 가장 중요
채널 인사보다 **먼저** 미끼를 던진다. 3요소를 이 순서로:
1. **충격 장면 선행** — 서사 순서를 무시하고 **본문 클라이맥스의 한 장면/대사를 통째로 맨 앞에** 꽂는다. (오리: "며느리가 비상금을 몽땅 들고 사라졌습니다. 끌고 온 것은 오리 새끼 천 마리." / 첩: "저 흉한 손으로 차린 밥상을 받으려니 밥맛이 떨어집니다.")
2. **결말 방향만 흘리기** — 결과를 스포하되 **정체·방법은 감춘다**. ("이 일이 농부의 팔자를 송두리째 뒤바꿔 놓을 줄은 아무도 몰랐습니다." / "이 비밀이 드러나는 순간 무사할 사람은 하나도 없었습니다.")
3. **질문으로 못 박기** — "과연 ~무엇일까요?" / "대체 왜 ~했을까요?"로 닫아 궁금증을 고정.

→ 인물 소개·배경 설명으로 시작하면 **안 된다**. 갈등의 정점부터 들이민다.

### ② 브릿지 (되감기) — 필수
콜드 오픈 직후, 두 단계로 본문에 진입:
- **채널 루틴 한 줄**: 인사 + 참여 유도 + "지금 바로 시작합니다". (야담 관습: "댓글에 건강 두 글자 남겨 주세요" 류 — 채널 색에 맞게.)
- **시간·장소 되감기 한 문장**: "조선 숙종 이십오 년 늦가을이었습니다" / "경기도 어느 고을에 장수라는 사내에게 시집온 며느리가 있었습니다." → 여기서부터 인물·결핍을 시간순으로 세운다. 콜드 오픈에서 스포한 장면은 본문 중반에 실제로 도착한다.

### ③ 본문 아크 — 결핍에서 응보까지
서사 비트를 아래 순서로 쌓는다. **길이에 따라 비트 수·디테일을 조절**(§4):
1. **결핍 세우기** — 주인공의 약점을 심는다: 가난·상실·장애·흉터·무자식·몰락. **결말 감정이 여기서 자란다.** 동시에 갈등의 씨앗(악인·유훈·떡밥)을 미리 깐다.
2. **사건/결단** — 이야기를 굴리는 첫 행동: 선행(쫓기는 이를 숨김) / 결단(비상금으로 오리 구매) / 발견(마루 밑 유서) / 이주(폐가 매입·가출).
3. **조력자 + 떡밥** — 조력자(신비한 노파·의로운 시아버지·늙은 소·어린 딸)를 붙이고, **정체가 나중에 밝혀질 미스터리 떡밥**을 심는다.
4. **갈등 계단식 상승** — 압박 주체를 **매 단계 키운다**: 개인 → 집단(마을 조롱) → 권력(관아·부호) → 배신(밀정·자객). "무시 → 학대 → 축출 → 누명"처럼 점층.
5. **최저점/파국** — 주인공이 가장 무너지는 지점. 착한 주인공이 공포에 굴복해 은인을 내치거나(자기 배반), 완전히 쫓겨난다. **이 바닥이 깊을수록 반전 카타르시스가 크다.**
6. **재기 + 복선 회수 시작** — 각성/자립. 초반에 심은 복선들이 하나씩 회수되기 시작. 이중 반전을 준비(믿었던 자가 흑막 / 노파가 조상 혼령).
7. **대반전 / 응보** — 정체 공개 + 악인 심판. 트리거는 외부의 힘(암행어사 출도 / 재앙 격퇴 / 관가의 증거 판정). 주인공이 직접 치지 않는다(§0 황금률).

### ④ 다중 해소 + 교훈 봉인
클라이맥스 후 바로 안 끝낸다. **보상을 연쇄로 쌓는다(reward stacking)** — 3~5연타: 재앙 격퇴 → 벼락부자 → 이웃 분배 → 정체 밝힘 → 임금 포상. 그다음:
- **주제문 직접 낭독으로 봉인**: "의로움이란 당장 손해처럼 보여도 끝내 가장 큰 복으로 돌아오는 법입니다." (감정축을 한 문장으로.)
- **여운 장치**: "오늘날까지 전해 내려옵니다"(전설화) 또는 콜드 오픈 이미지의 수미상관 회수(씨받이: "오백 냥의 무게가 사라진 허리는 허전했지만 발걸음은 단단했습니다").
- **채널 CTA**로 종료.

## 2. 리텐션 툴킷 (최소 3개 이상 박아라)

- **★"~는 아직 알지 못했습니다" 후렴** — 막이 끝날 때마다 다음 위기를 예고하는 클리프행어. 장편 기준 **5~6회 반복**이 표준. ("그러나 그 의로움의 값이 얼마나 혹독할지 그는 아직 알지 못했습니다.")
- **떡밥 반복 리마인드** — 핵심 미스터리(노파의 정체 / 안 쓰는 오백 냥 / 숨겨진 방)를 잊을 만하면 다시 꺼내 "왜?"를 갱신. 같은 묘사 문장을 거의 그대로 3~4회 재등장시켜도 좋다.
- **"바로 그때였습니다"** — 반전·조력자 등장·장면 전환 직전의 정형 트리거.
- **시점 교차(cross-cut)** — "한편, 시댁에서는…"으로 **흥하는 주인공 ↔ 망하는 악인**을 번갈아 붙여 대비 긴장. (긴 대본에서 특히.)
- **테마 대사 3~5회 반향** — 유훈·격언 한 줄("값을 따지지 않는 법이다")을 도입→결단→파국→각성→결말에 반복. 감정 각인 + 복선 이중 기능.
- **패턴 인터럽트**(장편 한정) — "잠시 숨 고르고 가실까요? 뒤로 갈수록 더 재밌어집니다" 류의 쉬어가기 + 다음 예고. 이탈 방지.

## 3. 복선 규칙 (절정을 값지게)

- **콜드 오픈의 미끼는 반드시 회수한다.** 던진 질문에 답하지 않으면 낚시로 끝난다.
- **복선은 초반에 촘촘히, 티 안 나게 심는다.** 회수 시 **토씨까지 같은 묘사**를 다시 써서 "아, 그거!"를 만든다. (유서 글씨: 첫 발견 "아래로 갈수록 흔들리다 마지막 줄은 알아볼 수 없이 비틀렸다" → 회수 시 동일 문장 = 독으로 죽어가며 쓴 손떨림.)
- **이중 반전을 노려라** — "믿었던 권위자가 진짜 흑막"(어진 현감=몸통) 또는 "조력자의 초자연적 정체"(노파=조상 혼령)로 한 번 더 뒤집는다.
- 정체 디테일은 **하나의 시각적 물건**으로 회수하면 강하다 (몸에 있는 유품·인장·악보·약재보따리).

| 5분(숏폼形) | ~1,200 | 3~4비트로 압축(결핍·사건·반전·해소) | 2~3회 | 2~3연타 |

## 5. 문체

- **종결어미: "~습니다" + "~지요/~것이었습니다/~것이랍니다" 구연체 교대.** "~습니다"로 사실을 진술하고 "~지요"로 정서를 얹는 2박자. **"~였다/~하였다/허나/~거늘" 문어체 낭독 금지**(사극 톤 = 거리감 = 조회수↓). "허나"→"하지만/그런데".
- **감정은 사물·행동 디테일로.** "슬펐습니다"라고 쓰지 말고 "짚신을 아궁이 앞에 데워 두었습니다"로. 절정·마무리일수록 담담하게 — 사실이 감정을 대신 옮긴다.
- **대사는 간접화법 위주, 결정적 순간만 직접 인용.** 직접 대사는 짧게(1~2문장), 현실 구어체 그대로(사투리·반말 허용): "저 사람, 실성했구먼." 단일 나레이션 전제이므로 대사도 나레이터가 자연스러운 호흡으로 읽는다(긴 웅변 금지).
- **시대는 특정, 지명은 모호** — 왕대는 구체적으로("조선 숙종 이십오 년"), 지명은 뭉갠다("경기도 어느 고을"). 대신 **조선 제도·경제 어휘로 사실감을 앵커**: 환곡·관기·군마·좌수·젓갈·정려·유배·선공감. 필요하면 현대어 각주 병기("역절풍, 요즘 말로는 류마티스 관절염이지요").
- **고유명사는 토속적 2음절** (분이, 돌이, 무던, 매심).
- 짧은 문장으로 리듬. 한 문장 = 한 동작/한 감정. 3문장 이상 같은 길이로 잇지 않는다.

## 6. 장편 운영 — 3~4만자 단일 대하 서사 (★주력 포맷)

§1~5의 골격은 길이 무관. 이 절은 3~4만자를 실제로 굴리는 **운영 규칙**이다. 단편(1만자 이하)은 이 절을 무시한다.

### 장(章) 설계 — OUTLINE 단계에서 확정

- 분량 배분: 공백 제외 30,000~40,000자 = **콜드 오픈+브릿지(800~1,200자) + 장 10~14개 × 2,500~3,500자** (장당 약 10~15분).
- 본문 아크 7비트 ↔ 장 매핑 (12장 기준 예):
  결핍 1~2장 / 사건·결단 3장 / 조력자·떡밥 4장 / 갈등 계단 5~7장 / 최저점·파국 8장 / 재기·복선 회수 9~10장 / 대반전·응보 11장 / 다중 해소·봉인 12장.
- **장 = 미니 아크**: 장 도입 1~2문장(새 의문/위협 제시) → 전개 → 장 끝은 **"~아직 알지 못했습니다" 후렴 또는 클리프행어**로 닫는다.
- 클리프행어 유형을 로테이션: **위협**(다가옴) / **발견**(단서) / **결단**(선택) / **도착**(인물·소식). 같은 유형 연속 금지.
- **중반 재점화 필수**: 전체 40~60% 지점(최저점 전후)에서 제2 미스터리 점화 또는 판 뒤집기 1회 — 장편 이탈이 가장 많은 구간. 시점 교차(§2)를 여기서 본격 가동.
- 패턴 인터럽트는 장 경계에만, 전체 2~3회.

### 복선 장부 — outline.md에 표로 유지

- 3~4만자에 복선 **4~7개**. 각 복선: `id | 회수 시 토씨까지 재사용할 묘사 문장 | 심는 장 | 리마인드 장 | 회수 장`.
- 절정에서 최소 2개 동시 회수. **회수 없는 복선 금지** (REVIEW에서 잡는다).
- **시각적 소품 복선 1개 이상 필수** — 이것이 ASSET_EXTRACT의 `anchorProp` 후보가 된다.

### 오디오/수면 시청 규칙 (귀로만 듣는 시청자가 다수)

- 후반부(대반전 이후) 새 인물 투입 금지.
- 인물은 이름으로 재언급 — "그는/그 사람" 연발 금지.
- 장 경계에서 한 문장 정리 허용 ("~은 그렇게 …했습니다") — 흐름을 놓친 시청자 복구용.

### 집필 운영 (DRAFT 단계)

- **2~3장씩 배치 순차 집필. 병렬 금지** — 연속성(문체·시간선·복선)이 깨진다.
- 스토리 바이블 `{S}/bible.md`를 배치마다 갱신: 인물 현황(위치·상태·아는 것), 시간선, 복선 체크오프, 테마 대사·후렴 사용 횟수, 다음 배치 주의사항.
- 집필 에이전트 입력 = 이 공식 + `script-constraints.md` + outline의 해당 장 브리프 + bible 최신본 + **직전 장 마지막 ~500자** (문체·연결 이어붙임용). **원전 자료(`_refs/`의 transcript·analysis)는 절대 포함 금지** — 표절·디테일 오염 차단 (SKILL "원전 격리").
