# 제안: STORYBOARD 분할 규칙 개정 (2026-07-19, 미적용 초안)

근거: `refs/storyboard_density_20260719/report.md` (상위 4채널 실측 — 밀도 곡선·트리거·재사용).
**이 문서는 제안까지만. SKILL.md 수정은 플랜 세션에서 거인 실험 세션 결과(shot 필드)와 병합 검토 후.**

## A. 씬 분할 기준 — 트리거 우선순위 명문화

현행 "인물 등장/퇴장, 장소·시간 전환, 사건·정서 전환"은 실측(우체국 트리거 상위 75%)과 일치 → 유지하되 우선순위와 신규 1종을 명시:

```
씬 경계 트리거 우선순위 (높을수록 반드시 끊는다):
1. 장소·시간 전환          (실측 ~35%)
2. 사건·발견 비트           (~25% — "바로 그때", 발소리, 문서 발견)
3. 인물 등장·발각           (~15%)
4. 클로즈업 인서트 ★신규    (~15% — 소품·신체 부위·리액션 손동작, 2~10초 단컷)
5. 감정 리액션              (~10% — 긴 대화 중 표정 전환)
```

- **클로즈업 인서트**: cast 없는 씬(소품/손/발만) — 캐릭터 일관성 리스크 0, ref 첨부 불필요. 발견·서스펜스 비트에만 아껴 쓴다(편당 5~10개). `insert: true` 표기, location 생략 가능.
- **홀드 허용 상향**: 회상·설명·이동 구간은 3~5문장 → **최대 8~10문장(3~5분)** 묶음 허용 (우체국 최장 319초, 월하는 전 씬 4.4분). 단 전 씬 Ken Burns 전제(→ D).

## B. 구간별 밀도 가이드 (분당 씬 수)

현행 "러닝타임(분)×0.8~1.1"(110분 → 90~120씬)은 시장 상단(몽이~우체국 수준). 실측 시장 범위는 **분당 0.35~1.0**이고 최고 성과(월하 47만)가 최저 밀도(0.22).

| 구간 | 제안 밀도 | 비고 |
|---|---|---|
| 훅 몽타주 (첫 60~90초) | 컷 10~15개 (4~8초/컷) | **신규 생성 0장** — 본문 씬 크롭 재사용(→ C) |
| 본문 | **분당 0.5~0.8** (75~150초/씬) | 씬 예산 = 러닝타임(분)×**0.5~0.8** (110분 → 55~90씬) |
| 절정·발각 구간 | 분당 1.5~2 (30~60초/씬) | 인서트 포함. 권장이지 필수 아님(월하는 평탄해도 1위) |

- 씬 예산 하향의 실익: flow 일일 쿼터(~45장/일) 기준 110분짜리 생성 기간 8일+ → **2~3일**.
- 지속시간 균등화(타이머 컷)는 계속 배격 — 서사 비트 기준 유지. 시장이 균등 컷으로도 성과 내는 건 Ken Burns 덕이지 균등함 덕이 아니다.

## C. 이미지 재사용 정책 — 훅 몽타주 전용 `image_ref`

실측상 본문 파일 재사용은 시장에도 없다(4편 합계 5건). 재사용의 실전 용처는 **훅 몽타주가 본문 이미지를 당겨쓰는 것** 하나 → 스키마도 그 용도로 한정 설계:

```jsonc
// storyboard.json 씬 스키마 확장 (예)
{ "id": 2, "act": "hook", "sentences": [1, 1],
  "image_ref": 47,                  // 본문 씬 47의 이미지를 재사용 (신규 생성 스킵)
  "crop": "face closeup",           // 옵션: CapCut/렌더 확대 힌트 (없으면 원본 그대로)
  "visual_desc": null }             // image_ref 씬은 visual_desc 불필요
```

- `build.py`: `image_ref` 있는 씬은 생성 스킵, 대상 씬 이미지 경로를 그대로 참조(검증: 참조 대상이 유효 씬 id이고 참조 사슬 금지 — ref의 ref 불가).
- **훅 시퀀스 구성 규칙**: 대본 훅 문장들(첫 막 요약·예고 성격)에 절정 장면 씬들의 `image_ref` + 크롭을 매핑. 씬1은 현행 유지(썸네일급 신규 생성 + VEO_HOOK 립싱크 클립) — 몽타주는 씬2부터.
- 본문 `image_ref`도 문법상 허용하되(재등장 정물·콜백 등) 기본은 씬마다 새 이미지(현행 유지).

## D. CapCut 마무리 가이드에 추가 (스킬 RENDER 절 or profile.md)
